## Objetivo do Projeto
 A criação de um sistema dinâmico desktop de cadastros de clientes e produtos, utilizando o Java Swing para uma navegação mais intuitiva entre as telas. Feito também para reforçar boas práticas e consolidar conhecimentos em POO, banco de dados e versionamento de código.

  ---

## Requesitos do projeto

-	RF001 – Cadastro de fornecedor: requisito responsável por cadastrar os fornecedores para a manutenção dos produtos do negócio, levando em consideração suas informações, como razão social, CNPJ, contatos e histórico de fornecimento.
  
-	RF002 – Cadastro de produtos: sistema responsável por inserir os produtos no estoque e no caixa.
  
-	RF003 – Identidade de produtos: sistema responsável por identificar cada produto de forma segura e precisa.
  
-	RF004 – Cadastro de clientes: aqui deve ser inserido os dados do cliente que poderão ser usados como formas de pagamento, como CPF, PIX, cartão de crédito e dinheiro, além de descontos caso siga as normas da empresa, como um clube próprio ou login no sistema.
  
-	RF005 – Caixa: o sistema deve conseguir usar a identidade de cada produto e informar seu preço e realizar o cálculo automático do valor total da venda.
  
-	RF006 – Forma de pagamento: aqui deve ser oferecido todas as formas de pagamento para o cliente e efetuar a compra.
